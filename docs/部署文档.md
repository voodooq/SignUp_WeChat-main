# 赛事报名系统 - 本地开发与部署文档

本项目现已完全迁移至 **FastAPI (后端) + MongoDB (数据库) + React (后台管理端) + UniApp (微信小程序前端)** 架构。为了方便进行本地开发与调试，我们提供了基于 Docker Compose 的打包与运行方案。

## 一、环境准备

在开始之前，请确保您的计算机已经安装了以下环境：
1. **Docker Desktop** 或 Docker 引擎
2. **Git**
3. **微信开发者工具**（仅用于小程序调试）

## 二、Docker 环境一键启动

我们用 `docker-compose` 来编排和启动 后端服务、后台管理端服务 和 数据库服务。

1. **进入项目根目录：**
   ```bash
   cd e:\work\SignUp_WeChat-main
   ```

2. **配置环境变量：**
   复制 `backend/.env.example` 为 `backend/.env`。并确保里面的 MongoDB URI 正确。如果您使用默认设置，无需修改：
   ```env
   # backend/.env 示例
   MONGO_URI=mongodb://sign-mongo:27017/sign_wechat
   ```

3. **构建并启动所有容器：**
   ```bash
   docker-compose up -d --build
   ```

4. **确认服务是否可用：**
   - **后端 API：** 访问 `http://localhost:8000/docs` （应能看到 FastAPI 的 Swagger 界面）
   - **后台管理系统：** 访问 `http://localhost:8080/` （应能看到 React 管理面板的登录页）
   - 管理员账号使用：可参考数据库初始化数据的说明，或者默认的超管设置。

> [!TIP]
> **重新构建**
> 如果您修改了 `backend/` 或 `frontend/` 下的代码，您需要重新构建镜像：
> `docker-compose up -d --build backend` 或 `docker-compose up -d --build frontend`

## 三、小程序端本地开发调试

由于小程序是在微信开发者工具或手机上运行的，它们无法像管理面板那样直接通过 Docker 内网或者反向代理解决请求问题。
请按照以下步骤配置并在本地调试：

1. **导入项目**
   在微信开发者工具中，选择项目根目录的 `miniapp/` （如果是分包或者直接对应目录，这里是 `miniapp`，但如果是 HBuilderX 架构，则导入对应的 UniApp 构筑包）。

2. **修改 BaseURL**
   定位到您的前端 API 配置文件，例如 `miniapp/src/services/api.ts` 或类似文件，将向后端的 API 请求根路径修改为局域网 IP 或者本地服务 `http://127.0.0.1:8000`。
   如果要在真机调试，必须将 URL 修改为您的内网 IP（例如 `http://192.168.1.100:8000`），注意真机调试下微信要求配置安全域名（或者在开发者工具内勾选 `“不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书”`）。

3. **编译运行**
   使用 HBuilderX （针对 UniApp 项目）重新发行到微信小程序，或者直接运行即可进行预览与真机联调。

## 四、数据初始化机制

新的架构中配置了 MongoDB 初始化脚本挂载：
当 `sign-mongo` 容器第一次启动时，它会自动读取根目录下的 `mongo-init/` 中的脚本并在数据库中录入初始的必备内容（如 `1000米`，`800米` 这类必考比赛等初始规则）。所以无需繁琐的手动建表步骤。

> [!NOTE]
> 如果想要重置数据库，可以删除本地 `docker volume` 挂载对应的卷，然后重新启动 Compose。
> `docker-compose down -v` （警告：此操作将清空所有参赛与系统配置数据！）
