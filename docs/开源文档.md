# 开源文档

本文档用于指导本项目对外开源发布，包含发布目标、信息边界、安全要求、使用指南和维护规范。

## 1. 项目定位

- 项目名称：赛事报名微信小程序 (私有化部署版)
- 项目类型：微信小程序 + FastAPI + MongoDB + Docker
- 核心场景：赛事报名、缴费、准考证、管理后台、成绩管理
- 目标用户：学校/机构赛事组织方、学生参赛用户、管理员

## 2. 开源范围

- 小程序端代码（`miniapp/`）
- 后端 FastAPI 业务逻辑（`backend/`）
- React 管理后台逻辑（`frontend/`）
- 容器化部署脚本（`docker-compose.yml`, `Dockerfile`）

建议保留私有或脱敏后再公开：

- 小程序 `AppSecret`、支付商户密钥、回调密钥
- 真实生产环境回调地址、数据库连接敏感配置
- 真实用户数据、订单数据、身份证明数据

## 3. 发布前安全检查清单

发布前请逐项确认：

- [ ] 已替换/清理所有密钥（`appid` 除外，`secret` 必须清理）
- [ ] 未提交真实商户号密钥与支付签名密钥
- [ ] 未提交任何真实用户数据导出文件
- [ ] 示例配置文件使用占位符（如 `YOUR_WECHAT_SECRET`）
- [ ] 云函数中未硬编码敏感密钥（统一走环境变量/配置中心）
- [ ] README 与文档中不包含可直接复用的生产凭证

## 4. 快速开始（开源版）

### 4.1 环境准备

- Docker & Docker Compose
- Node.js 18+ (用于本地开发小程序)
- HBuilderX（最新版）
- 微信开发者工具

### 4.2 安装与运行

```bash
# 启动全栈环境 (后端 + 数据库 + 管理后台)
docker-compose up -d --build
```

然后通过 HBuilderX 运行 `miniapp` 到微信开发者工具进行本地联调。

### 4.3 数据库初始化

- 数据库表结构会自动由后端程序在启动时初始化。
- 基础赛事数据可通过管理后台进行录入。

## 5. 配置规范

建议使用 `uni-config-center`：

- `sign-user/config.json`
- `sign-payment/config.json`

使用占位符示例：

```json
{
  "wechat": {
    "appid": "YOUR_APPID",
    "secret": "YOUR_APP_SECRET",
    "mchId": "YOUR_MCH_ID",
    "apiKey": "YOUR_API_KEY",
    "notifyUrl": "https://your.domain/path",
    "subscribeTemplateId": "YOUR_TEMPLATE_ID"
  }
}
```

## 6. 功能与权限模型

- 普通用户：报名、查询、支付、查看准考证
- 管理员：项目管理、报名管理、扫码查询、图片管理、手动报名
- 权限判定：以云函数查库结果为准，不依赖前端角色字段

## 7. 数据与隐私说明

- 身份证号等敏感信息应加密存储
- 前端默认只展示脱敏字段
- 管理员读取完整信息需要服务端鉴权
- 建议在仓库补充隐私声明与数据处理说明（可新增 `PRIVACY.md`）

## 8. 贡献指南（建议）

欢迎提交 Issue 与 PR。建议流程：

1. Fork 并创建功能分支
2. 小步提交，保持可读的 commit message
3. 自测核心流程（登录、报名、支付回调、管理员查询）
4. 提交 PR 并说明变更动机、影响范围、测试结果

可在仓库中新增：

- `CONTRIBUTING.md`
- `CODE_OF_CONDUCT.md`
- `SECURITY.md`
- `LICENSE`（若与当前 `ISC` 不同需同步调整）

## 9. 版本发布建议

- 使用语义化版本（`MAJOR.MINOR.PATCH`）
- 每次发布附带 `CHANGELOG`
- 关键功能变化（支付、权限、数据结构）需在 release note 明确标注

## 10. 已知限制

- 未完成企业认证时，部分微信授权能力（如手机号、头像昵称）受限
- 支付与订阅消息依赖微信侧配置及资质
- 线上部署需结合业务场景完善风控与审计日志
