# 赛事报名微信小程序 - 项目说明

## 一、项目概述

基于 **FastAPI (后端) + MongoDB (数据库) + React (后台管理端) + UniApp (微信小程序前端)** 的全栈架构，用于体育赛事报名、缴费、准考证生成及后台管理。

### 技术栈

| 技术 | 说明 |
|------|------|
| UniApp (Vue3) | 移动端前端框架，编译为微信小程序 |
| TailwindCSS 3.x | 样式方案，通过 `npx tailwindcss` 编译到小程序端 |
| React + Vite | Web 独立后台管理端，提供完善的赛事面板操作 |
| FastAPI | Python 高性能后端框架，替代原先的云函数 |
| MongoDB | 统一非关系型数据库，替代原有基于阿里云的方案 |
| Pinia + persistedstate | 状态管理与持久化 (小程序端) |
| 微信支付 | 小程序内支付及回调 (通过 FastAPI 解密和验证) |

### 注意事项

- 本版本已全盘剥离了以往关联的阿里云 `uniCloud-alipay/` 等冗余云端云函数。
- 小程序现在仅是一个纯粹的 View 和少量的业务层组合，统一向 FastAPI 后端的暴露接口发送请求。
- FastAPI 对数据库访问使用了 ORM 及原生的 MongoDB 连接，所有 Schema 定义存在于项目后端目录。
- API 服务包含统一错误处理中间件及鉴权逻辑。

## 四、后端 API 设计（FastAPI）

### 4.1 `/api/auth` 用户认证相关

| Action路由 | 说明 |
|--------|------|
| `/api/auth/login` | 微信登录，通过前端获取的 code 请求后端。后端请求微信并下发 JWT Token，创建/更新用户记录 |
| `/api/auth/me` | 获取当前用户信息及角色，鉴权：验证 JWT Token |

### 4.2 `/api/register` 报名相关

| Action路由 | 说明 |
|--------|------|
| `/api/register/submit` | 提交报名表单（校验身份证格式、手机号格式、是否重复报名），生成准考证号 |
| `/api/register/check` | 检查是否已报名该项目 |
| `/api/register/my`| 获取当前用户所有报名记录 |
| `/api/register/ticket/{ticket_no}`| 根据准考证号获取准考证信息 |

**防重复报名逻辑：**
1. 前端提交前先调用校验接口
2. 后端核心路由内部事务再次校验确认

### 4.3 `/api/payment` 缴费相关

| Action路由 | 说明 |
|--------|------|
| `/api/payment/create` | 创建微信支付订单，调用微信支付统一下单API，返回支付参数给前端 |
| `/api/payment/notify` | 微信支付回调通知，验签后更新报名记录的缴费状态 |
| `/api/payment/query` | 主动查询订单支付状态（兜底机制） |

**安全性：**
- 支付金额由服务端数据库记录读取，不信任前端传入的金额
- 回调通知验证微信签名，防止伪造
- 订单号全局使用UUID唯一化设计

### 4.4 `/api/admin` 管理员相关

| Action路由 | 说明 |
|--------|------|
| `/api/admin/registrations` | 会签带查询参数检索所有用户的报名记录 |
| `/api/admin/registrations/{id}`| 对特定报名的调整/手动修缮 |
| `/api/admin/banners` | 首页轮播图的增删改查 |
| `/api/admin/events` | 比赛项目管理（增删改查） |

**所有 `/api/admin/*` 管理员接口必须在中间件或前置依赖中校验当前 JWT 用户包含 admin 标识，否则返回 HTTP 403。**

## 五、核心功能详细设计

### 5.1 首页（参考设计图）

**布局从上到下：**
1. **顶部轮播图** — 从 `sign-banners`（position=top）读取，管理员可自定义上传
2. **报名倒计时** — 从 `sign-settings` 读取截止时间，前端倒计时显示
3. **功能按钮第一排** — 「立即报名」「报名查询」「查询成绩」
   - 立即报名 → 静默登录 → 参赛须知确认页 → 报名表单
   - 报名查询 → 静默登录 → 报名记录列表（result.vue mode=list）
   - 查询成绩 → 成绩查询功能（待开发）
4. **功能按钮第二排**（仅管理员可见）— 「管理后台」→ 跳转 `pages/admin/index`
5. **赛事信息区域** — 展示赛事详情
6. **底部轮播图** — 从 `sign-banners`（position=bottom）读取

### 5.2 报名流程

```
用户点击「立即报名」
    ↓
检查登录状态（未登录则静默调用 uni.login 自动登录）
    ↓
跳转「参赛须知确认页」（register/confirm.vue）
  - 展示比赛说明图片（/static/race-notice.png）
  - 底部按钮默认禁用，5秒倒计时后启用
  - 需勾选"我已阅读并同意以上参赛须知"
  - 点击「确定」→ 跳转报名表单页
    ↓
展示报名表单：
  - 姓名（必填）
  - 性别（picker 选择：男/女）
  - 所属学校（必填）
  - 电话（必填，校验手机号格式）
  - 身份证号（必填，校验18位身份证格式）
  - 必考项目（根据性别自动选定：男→1000米，女→800米）
  - 选考项目（多选：掷实心球、立定跳远、1分钟跳绳、坐位体前屈）
    ↓
提交报名 → 云函数生成准考证号
    ↓
跳转缴费页 → 微信支付
    ↓
缴费成功 → 结果页（显示准考证号，可查看准考证）
```

**已报名用户：** 跳转到报名记录页，展示已报名信息和准考证。

### 5.2.1 必考与选考项目规则

| 类型 | 男生 | 女生 |
|------|------|------|
| **必考** | 1000米 | 800米 |
| **选考** | 掷实心球、立定跳远、1分钟跳绳、坐位体前屈 | 掷实心球、立定跳远、1分钟跳绳、坐位体前屈 |

- 必考项目根据性别自动选定，不可更改
- 选考项目可多选，男女通用，**最多选择2项**
- 报名记录存储字段：`required_event`（必考）、`optional_events`（选考数组，最多2项）、`event_name`（汇总文本）

### 5.3 身份证号安全存储

- 后端接收到身份证号后，使用 `crypto-js` 或等效 Python 工具做 AES 加密后存入数据库
- 查询时在服务端内解密，仅返回脱敏后的身份证号（如 `3201**********1234`）给前端
- 管理员查询时可返回完整身份证号（需管理员权限校验）
- 加密密钥存储在后端环境变量中，不硬编码

### 5.4 准考证与条形码

**准考证页面设计（参照实体准考证样式）：**
- 顶部标题：「2026年体育赛事 准考证」
- 信息表格：考点、准考证号、姓名、性别、报名点、报考项目
- **条形码**：使用 Code128B 编码，canvas 绘制，显示准考证号
- 底部提示：「请妥善保管准考证，比赛当天凭此入场」

**条形码组件**：`components/Barcode.vue`
- 基于 canvas 的 Code128B 条形码生成器
- 支持自定义宽高、是否显示文字
- 管理员可通过 `wx.scanCode()` 扫描条形码查询

### 5.5 管理员扫码查询

**实现方案：**
1. 准考证页面生成包含 `ticket_no` 的 Code128 条形码
2. 管理员进入扫码页面 → 调用 `wx.scanCode()` 扫描条形码
3. 解析准考证号 → 调用 `searchRegistration` 查询报名信息
4. 展示完整报名信息（含完整身份证号）

## 六、管理员与普通用户区分方案

### 方案：数据库角色字段 + 云函数鉴权

**实现步骤：**

1. **`sign-users` 表增加 `role` 字段**
   - 默认值 `user`（普通用户）
   - 管理员设为 `admin`

2. **初始管理员设置**
   - 首次部署时，通过 uniCloud Web 控制台直接在 `sign-users` 表中将指定用户的 `role` 改为 `admin`
   - 或在 `sign-settings` 表中配置 `admin_openids` 白名单，首次登录时自动赋予管理员角色

3. **前端路由守卫**
   - 用户登录后，Pinia store 缓存用户角色
   - 访问 `pages/admin/*` 页面时，`onLoad` 中校验角色，非管理员跳转回首页
   - TabBar 或个人中心页面根据角色动态显示「管理后台」入口

4. **后端鉴权（核心安全）**
   - 所有 `/api/admin` 路由的中间件内部：
     ```python
     # 1. 从 JWT Token 中解析并提取用户信息和 role 标识
     # 2. 如果 role 并不等于 'admin'，则抛出 HTTPException
     if user.role != 'admin':
         raise HTTPException(status_code=403, detail="无管理员权限")
     ```
   - **绝不信任前端传入的角色信息**，每次请求都通过 token 与数据库映射查实

5. **管理员功能入口**
   - Web 版管理员界面，登录后包含：图片管理、报名查询、扫码查询、项目管理、手动报名

## 九、关键安全措施

1. **身份证号加密** — AES 加密存储，密钥在云函数环境变量
2. **防重复报名** — 服务端 openid 唯一性校验（每人仅允许一条报名记录）
3. **支付安全** — 金额服务端读取，回调验签，订单号唯一
4. **管理员鉴权** — 云函数内部每次查库校验角色，不信任前端
5. **openid 获取** — 通过 `wx.login()` + 云函数 `code2Session`，不在前端暴露 secret
6. **接口防刷** — 云函数内增加频率限制（同一 openid 短时间内不可重复提交）
