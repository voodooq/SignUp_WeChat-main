# 🚀 赛事报名系统 - 第一阶段（后端迁移与前端核心改造）总结

本项目已成功从原有的 `uniCloud` Serverless 架构迁移至标准化的 `FastAPI + MongoDB` 后端架构，并实现了小程序核心业务逻辑的对接。以下是本次改造的核心成果与技术细节：

## 1. 架构升级与安全加固

### 🛡️ 身份认证大重构
* **彻底摒弃前端传递 OpenID**：原系统在每次调用云函数时，由前端传递 `openid`，存在严重的**越权操作风险**（恶意用户可通过修改传递的 openid 篡改他人数据）。
* **引入 JWT (JSON Web Token)**：后端新增 `/api/user/login` 接口，前端发送 `code`，后端配合微信接口换取 `openid` 并签发具有过期时间限制的 JWT Token。前端存储该 Token 并在后续 HTTP 请求的 Authorization 头中携带。
* **基于依赖注入鉴权**：借助 FastAPI 的 `Depends` 依赖注入体系，优雅实现接口级鉴权，杜绝越权访问漏洞。

### 🔒 隐私数据加密
* **身份证加密存储复刻**：在新的 `crypto.py` 中实现了标准 AES-256-CBC 加密逻辑，确保身份证等敏感信息在 MongoDB 中密文存储，防止数据库泄漏导致用户隐私泄露。
* **哈希防重**：增加了基于 SHA256 加盐不可逆哈希算法的 `id_card_hash` 字段，配合唯一索引，实现了既保护明文又能高效判断重复报名的需求。

### 📁 文件上传安全
* **完全剥离云存储**：重写了基于本地服务的文件上传 `/api/upload`。
* **防枚举攻击**：废弃了原有的时间戳命名规则，改用高强度随机文件名。
* **文件类型与大小限制**：在 FastAPI 层面拦截非图片文件及超大文件上传。

---

## 2. 数据库设计与容器化部署

* **Motor 异步驱动**：后端采用官方推荐的 `motor` 纯异步驱动。
* **Docker 快速编排**：设计了 `docker-compose.yml` 方案，可同时启动 `backend` 与 `mongo`。
* **自动初始化机制**：`mongo-init/init.js` 脚本不仅会自动创建所有复杂的数据库索引，还会自动写入必考项目及初始系统设置。

---

## 3. 前端通信层重构（uniCloud => HTTP）

原先分散在数十个 vue 页面中的 `uniCloud.callFunction()` 逻辑现已被模块化、标准化的 API 服务替代。

* **全新请求统一封装**：创建了 `services/api/request.js`。
  * 实现了基于标准 `uni.request` 的封装，自动维护与携带 JWT Token。
  * 无缝拦截 `401 / 403 / 422` 状态码进行统一的无权跳转和静默登录。
* **核心流程对接跑通**：
  * **登录态 (`user.js`)**：使用 API 彻底替代了原先复杂的云函数认证流程。
  * **报名表单与准考证 (`register.vue`、`ticket.vue`)**：全部转为基于 HTTP 的数据交互。
  * **管理端 (`admin/*.vue`)**：实现了调用代码的自动化改造与重点逻辑对接（注：复杂的管理需求下一步将由独立 Web Dashboard 接管）。

> **TIP：API 地址配置化**
> 按照您的需求，在 `services/api/config.js` 中设定了默认地址 `${DEFAULT_BASE_URL}`，方便随时连接本地开发环境（如 localhost:8000）或部署后的线上服务器域名！
