## 项目初始化

目前数据库会自动在第一次运行 `docker-compose up` 时根据根目录下的 `mongo-init/01_init.js` 脚本土动注入表数据和索引，因此无需手动往 `sign_wechat` 数据库里录入基础的项目信息。

如果您有额外的比赛项目需要初始化录入，可以编辑或覆盖 `mongo-init/01_init.js` 文件内的 `db.sign_events.insertMany` 数组部分，然后在构建新容器前执行 `docker-compose down -v` 来重新初始化。

## 成绩导入 Excel 模板

建议列头（第一行）：

```text
准考证号,姓名,项目名称,成绩,名次,备注,排名
```

示例：

```text
2026010001,张三,1000米,3'45",1,,1
2026010002,李四,1000米,3'52",2,,2
2026020001,王五,立定跳远,2.35m,1,,1
2026020002,赵六,立定跳远,2.22m,2,,2
2026030001,孙七,1分钟跳绳,156次,1,,1
2026030002,周八,1分钟跳绳,149次,2,,2
```

说明：

- `成绩` 保持原始展示格式（如时间/米/次数）。
- `排名` 为最后一列，用于同项目内统一排序，**数值越小排名越靠前**。
- 如果不同项目计量单位不同（秒、米、次），建议由管理员在导入时手工填写 `排名` 作为最终排序依据。

## 缴费成功微信通知（订阅消息）

已支持：用户缴费成功后自动向对应微信用户发送服务通知（订阅消息）。

### 小程序通知模板字段（当前默认映射）

```text
赛事名称   {{thing1.DATA}}
赛事地点   {{thing2.DATA}}
参赛人员   {{name3.DATA}}
开始时间   {{date4.DATA}}
备注       {{thing7.DATA}}
```

默认模板ID：请在你自己的微信小程序后台申请后填写，文档不提供真实值。

### 推荐：使用本地 `.env` 环境变量配置（新版配置方式）

我们现在的后台使用 FastAPI 架构，相应的配置全部迁移到了 `backend/.env`。请将密钥等敏感信息在此文件录入而不是写在前端或数据库。

#### 1）配置目录

```text
backend/.env
```

#### 2）配置内容

在 `backend/.env` 中新增或者维护（参考 `.env.example`）：

```text
WECHAT_APPID=YOUR_WECHAT_APPID
WECHAT_APP_SECRET=YOUR_WECHAT_APP_SECRET
WECHAT_MCH_ID=YOUR_WECHAT_MCH_ID
WECHAT_API_KEY=YOUR_WECHAT_API_KEY
NOTIFY_URL=https://your-domain.com/api/payment/notify
SUBSCRIBE_TEMPLATE_ID=YOUR_SUBSCRIBE_TEMPLATE_ID
```

#### 3）部署生效机制

修改 `.env` 后，如果位于 Docker 容器内，您需要重启容器：
```bash
docker-compose restart backend
```
如果是本地 python 运行环境则直接重启即可生效。

#### 4）可选：`sign-settings` 业务文案配置（不含密钥）

可在 `sign-settings` 中配置以下业务文案（用于通知内容展示）：

```text
event_title      赛事名称
event_location   赛事地点
event_date       比赛时间
```

### 发送内容

- 比赛项目（event_name）
- 赛事地点（location 文案）
- 参赛人（name）
- 开始时间（event_date）
- 备注（固定提示语）
